# 交互深度规范（点击深度审计）

> 日期：2026-07-22
> 适用范围：水质运维系统 PC端 + 移动端全部核心流程
> 核心目标：任何按钮、关联数据、状态，在合理点击次数内可达，避免过度点击或埋得太深

---

## 一、核心原则

### 1.1 不追求绝对点击数，追求"每一步都有明确预期"

"三次点击"是参考线而非铁律。真正的标准是：**每一次点击都有清晰预期、有信息反馈、不让人迷路**。一个4步但每步都清晰的操作，好过一个2步但让人猜的操作。

### 1.2 按用户场景区分严格度

| 用户 | 场景 | 严格度 |
|------|------|--------|
| 一线运维（移动端） | 戴手套、户外强光、频繁打断 | **最严格**：高频动作1-2次触达 |
| 管理者（PC端） | 坐着审核、低频深度操作 | 适中：3-4步可接受，信息渐进展开 |

### 1.3 按操作频率分级设定深度预算

| 操作频率 | 触达深度 | 典型操作 |
|---------|---------|---------|
| 每天多次（核心） | ≤2次触达 | 看今日任务、打卡、拍照、提交检查项、接单 |
| 每天/每周（常规） | ≤3次点击 | 排周计划、确认告警、工单处置、审批待办 |
| 偶尔（配置/深度） | ≤4次点击 | 规则配置、报表导出、台账维护、用户管理 |

```
┌─────────────────────────────────────┐
│  顶部：页面标题 + 状态提示            │

│                                     │

│                                     │
├─────────────────────────────────────┤
│  底部固定操作区（拇指区）：            │
│  - 主按钮 88rpx，永远在拇指可达处     │
│  - 打卡/拍照/提交不藏在页面中部       │
├─────────────────────────────────────┤
│  TabBar：首页|巡检|工单|告警|我的     │
│  （≤5个tab，一级直达，无二级菜单）     │
└─────────────────────────────────────┘
```

**关键规则：**
- 底部面板（Bottom Sheet）替代多层页面跳转，最多一层蒙层
- 状态即入口：首页"待办工单 3"数字可点，直达工单列表
- 异常项标记后自动弹相机，减少"找上传按钮"的步骤

```
│ 站点管理  │  │  └ 点击行→右侧抽屉详情          │  │
│ 物资管理  │  │     （关联数据分tab同屏）        │  │
│ 告警中心  │  └────────────────────────────────┘  │
│ 审核中心  │                                      │
│ 影像查询  │  抽屉：详情+操作+关联数据，关掉回列表   │
│ 报表分析  │  （不丢失浏览位置）                    │
│ 系统设置  │                                      │
└──────────┴──────────────────────────────────────┘
```

**关键规则：**
- 菜单最多两级，任何功能从侧边栏≤2级可达
- 列表行内放高频操作（通过/退回/查看），不强迫进详情页
- 详情用侧边抽屉（Drawer）而非跳转新页面
- 关联数据同屏聚合（审批工单时，告警/巡检/站点情况分tab展示）
- 深度页面必须有面包屑，可跳回任意层级

## 四、反模式清单（禁止）

| 反模式 | 危害 | 正确做法 |
|--------|------|---------|
| 高频动作埋进"设置"或"更多"菜单 | 找不到，放弃使用 | 高频动作放在主路径 |
| 连续弹窗（弹窗里再开弹窗） | 迷失上下文 | 最多一层蒙层，用底部面板/抽屉 |
| 跳转后丢失上下文 | 回来要重新筛选翻页 | 抽屉/返回保留筛选状态 |
| 关键状态要点进去才看见 | 增加无谓点击 | 状态在列表/卡片直接露出 |
| 同一信息散落多个页面 | 拼凑信息，效率低 | 关联数据同屏聚合 |
| 提交时才报错（如照片不足） | 挫败感，返工 | 过程中实时提示缺什么 |
| 操作按钮在页面中部/顶部（移动端） | 拇指够不到 | 主操作固定底部拇指区 |
| 审批必须进详情页再返回 | 连续审批效率低 | 行内快捷操作+抽屉 |

## 五、验收标准

每个页面/流程上线前，按以下清单验收：

1. **核心流程深度达标**：对照本规范第二/三节审计表，实际点击路径不超过标注深度。
2. **每步有反馈**：每次点击后有明确的状态变化或信息展示，无"点了没反应"。
3. **无死胡同**：任何页面都有明确的返回/下一步路径，不卡死。
4. **状态外露**：列表/卡片直接显示关键状态（待审/已审/异常/缺拍），不靠点进去看。
5. **上下文保留**：从详情返回列表，筛选条件和滚动位置不丢失。
6. **移动端拇指可达**：主操作按钮在底部，单手可完成核心流程。
