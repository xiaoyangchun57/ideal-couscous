核心流程 · 步骤端分布

🔄 被动响应链路（数据异常 → 工单关闭）

数据接入·异常检测 → 告警生成·分级 → 自动派单(按站点归属) → 移动端确认/接单 → 现场处置·拍照 → 核验(双端) → 告警关闭·审核归档

📅 主动计划链路（周计划 → 执行）

运维排程+智能建议 → 车辆/备件确认 → 提交审核 → 管理员审核(PC) → 自动流转(任务/车/备件) → 移动端执行 → 审核(双端)

🔍 巡检全链路（规划 → 执行 → 质控）

模板/计划配置(PC) → 计划审批(PC) → 到站打卡(GPS) → 逐项填报+拍照 → 提交 → 质控审核(双端) → 异常自动转告警

🧪 试剂更换链路（更换 → 质控 → 恢复）

试剂主数据/库存(PC) → 现场更换登记(待新增) → 质控验证·跑标样(待新增) → 通过→更新剩余天数 / 不通过→校准/报修(PC)

🔧 备件链路（出库 → 使用 → 回收）

计划确认备件需求 → 审批通过自动出库 → 库存扣减·记录(PC) → 现场使用·关单确认(待新增) → 旧件回收拍照(待新增) → 维修/报废(PC·P3)

当前缺口与建设重点（按优先级）

  • 衔接层缺失（高）：被动层产生的工单/告警没有自动流入周计划建议，"巡检迎合工单"的智能化价值未释放——这是当前最大断点。
  • 移动端周计划入口缺失（高）：周计划目前仅PC可新建/审批，运维人员（高频移动端用户）无法在手机上排程、确认资源、提交审核。
  • 移动端试剂/备件操作缺失（中高）：试剂更换登记、质控验证、备件使用确认、旧件回收都在移动端无入口，现场动作无法闭环。
  • SPC动态阈值未落地（中高）：spc_3sigma 仅有骨架，仍依赖固定/手动阈值（"太简单太粗暴"），需真实数据积累后落地。
  • 数据模拟器关闭（部署项）：当前静态演示模式，真实部署需对接实际数据源，被动层才能运转。
  • 移动端无数据审核入口（中）：L3人工复核仅PC可做，管理者移动审批时无法处理数据审核类待办。
总体判断：系统的被动智能层是强项（异常检测、自动派单、三级审核、SLA升级、闭环关联都已实现），主动计划层和衔接层是短板。两端职责清晰——PC端=管理/配置/规划/台账/报表中心，移动端=现场执行+移动审批中心，审批是唯一深度重叠域。下一步核心不是重建被动层，而是补齐衔接层（让工单/告警自动喂入计划建议）和移动端计划/物资操作入口，把已有的智能化能力真正串联成闭环。
