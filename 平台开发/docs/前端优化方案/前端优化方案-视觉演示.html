设计说明（对应方案文档 §4 小程序端）

① 待办聚合卡：把"要做什么"前置，替代原来 9 个平铺入口；每卡深链直达处理页。

② 任务流按路线排序：巡检计划迎合工单（顺路处理），卡片上直接体现"顺路工单"。

③ 出车确认前置：接单后、出发前落实车辆，符合实际作业流程。

④ 令牌统一：品牌色 #0D9488（浅）/ #00C9A7（深）、状态四色、圆角 8/12/16、间距 4 的倍数——全部收敛到 app.wxss，组件不再各写各的颜色。

⑤ 触达效率：打卡/拍照/接单 1–2 次点击；任何状态 3 次内可达。

说明：以下为双端共享的设计令牌与组件规范。PC 端通过 Antd5 ConfigProvider theme.token 落地， 小程序端通过 app.wxss CSS 变量落地，同一语义、同一取值。点击色块可复制色值。

双端落地映射

同一套令牌在两端的具体落点——改一处变量即全局生效，这是"统一"能低成本落地的关键。
令牌PC 端（React + Antd5）小程序端（WXSS）
品牌色ConfigProvider → token.colorPrimary（已有 useTheme，补全 token 集）app.wxss page { --brand: #0D9488 }（替换 #2B6CFF）
状态色token.colorSuccess/Warning/Error/Info + StatusTag 组件--success / --warning / --error / --info + 公共样式类 .stag-*
字号/间距/圆角token.fontSize / borderRadius / margin--fs-* / --sp-* / --r-* 变量，组件引用变量
