# 水质智慧运维系统 · 核心目标体系（V1.1）

> 定位：驱动整个项目优化、迭代与健康的唯一指导性目标。任何新功能、新迭代必须回答"它移动了下面哪个指标"，答不上就不排期。
>
> 说明：本版本不设具体目标值。所有指标先埋点采集基线，跑出真实数据后再逐项填入目标值（见"五、运转机制"）。指标的**定义、公式、方向**即为当前的考核依据：方向对了就是进步，方向反了就是退步。

## 〇、一句话核心目标

**让三类用户在绝大多数关键运维动作中主动通过本系统完成工作：一线运维用一个小程序无压力贯穿全程，审核员摆脱材料审核压力并一键出报表，管理者实时掌握所有人员的安排、进度与运维效果。**

## 一、北极星指标（唯一终极指标，季度考核）

**关键运维动作线上闭环率 —— 持续上升，直至线上成为默认工作方式**

计算公式：`线上闭环率 = 通过系统完成并闭环的关键动作数 ÷ 实际发生的关键动作总数`

四个可算子项（分母全部可从系统内取得）：

| 子项 | 公式 | 分母来源 |
|---|---|---|
| 巡检覆盖率 | 系统内完成巡检数 ÷ 周计划应巡数 | 周计划（每周每站必巡，应巡数固定已知） |
| 工单线上闭环率 | 系统内走完 接单→处置→审核→关单 的工单数 ÷ 全部工单数 | work_orders 表 |
| 告警处置线上率 | 系统内处置的告警数 ÷ 全部告警数 | alerts 表 |
| 审核线上完成率 | 系统内完成的审核数 ÷ 全部待审项 | audit/pending 聚合 |

含义：不用系统，动作就发生在系统外，此率自动下降——它直接编码了"想起来且愿意用"。

## 二、角色主指标（领先指标，月度考核）

### 1. 一线运维（诉求：一个小程序贯穿始终、无使用压力）

| 指标 | 方向 | 公式 / 说明 |
|---|---|---|
| 单站巡检耗时中位数 | 持续下降 | 打卡时间 → 该站最后一项提交时间 |
| 一次提交通过率 | 持续上升 | 首次提交即过审数 ÷ 总提交数（被驳回=分子不计） |
| 核心动作触达深度 | 越浅越好 | 打卡/拍照/接单从首页起的点击数（交互深度规范已有审计法） |
| 离线闭环成功率 | 趋近全部成功 | 断网期间完成并最终成功同步的巡检闭环数 ÷ 断网闭环总数 |
| 主动发起占比 | 持续上升 | 主动上报（拍照上报/即席工单/备件申请）÷ 全部动作——"愿意用"的最强信号 |

### 2. 审核员（诉求：减压 + 报表直出）

| 指标 | 方向 | 公式 / 说明 |
|---|---|---|
| 单条审核耗时 | 持续下降 | 打开待审项 → 提交结论；异常驱动审核落地后应大幅下降 |
| 待审清空周期 | 持续缩短 | 待审项创建 → 审结的 P90 时长 |
| 报表自助直出率 | 持续上升 | 系统直接导出的报表数 ÷ 全部报表需求数（手工拼 Excel=失败） |
| 异常识别召回率 | 保持高位不滑坡 | 抽样复核：自动通过的项中真实异常漏判比例——防止"减压"变"漏审"。此项是审核减压的对冲指标：耗时降但漏判升 = 判负 |

### 3. 管理员（诉求：掌握所有人的安排、进度与效果）

| 指标 | 方向 | 公式 / 说明 |
|---|---|---|
| 进度可见时延 | 持续缩短 | 一线动作发生 → 管理端可见 |
| 计划 vs 实际可视率 | 趋近全覆盖 | 任一人/任一站，本周计划与实际执行对比一屏可见 |
| 行动队列决策占比 | 持续上升 | 管理者审批/处置动作中从行动队列入口发起的比例 |
| 站点健康度月度改善 | 可量化、可追因 | 各站健康度环比趋势可导出、可追因到人/到事 |

## 三、护栏（持续监控的底线，不进 KPI）

- 关键接口响应时长不劣化（以基线为准，持续监控 P95）
- 主流分辨率/手机机型无布局破损（多端适配归护栏，不是目标本身）
- 前端错误率、后端 5xx、可用性不劣化（以基线为准）
- **架构债不恶化**：app.py 冻结净增量，新功能走新模块；不为"行业前沿"重写框架

## 四、反目标（防止指标被玩坏）

- **不追 DAU、不追使用时长、不追功能数量。** 核心是"减负还愿意用"，耗时越短越好，与 DAU 天然相反。
- 任何新功能立项必须回答："它抬升哪个角色主指标？"答不上就砍（功能消失测试）。
- 禁止为凑闭环率强制线下动作搬线上——以角色耗时指标为对冲：闭环率升但耗时也升 = 反向增负，判负。

## 五、运转机制

1. **节奏**：北极星按季度复盘；角色主指标按月复盘；护栏持续监控告警。
2. **排期规则**：每个迭代必须挂钩至少一个角色主指标的移动，否则不排期。
3. **目标值产生方式**（对应任务 #318）：先埋点采集——单条审核耗时、主动发起占比、单站巡检耗时、一次提交通过率等——跑 2~4 周取得基线，再按"基线 + 合理改善幅度"逐项填入目标值，形成 V2.0。**在基线出来之前，本文件不含任何数字目标，只考核方向。**
4. **指标冲突裁决顺序**：护栏 > 角色主指标 > 北极星。先保底线，再保体验，最后冲覆盖率。
